1. Concept sản phẩm đúng

LemonInk không chỉ là một website chứa các bản tóm tắt sách có sẵn. Ý tưởng chính là dùng AI để xử lý sách: trích xuất nội dung, tóm tắt, tạo audio từ bản tóm tắt và hỗ trợ chat hỏi đáp theo nội dung sách. Thư viện của người dùng sẽ gồm cả sách có sẵn từ LemonInk và sách cá nhân do người dùng upload.

Kho sách có sẵn của LemonInk

Đây là thư viện public do admin chuẩn bị hoặc do hệ thống xử lý trước. Người dùng có thể khám phá, đọc, nghe audio và chat ngay.

  • Hiển thị trong thư viện chung.
  • Có thể được admin biên tập.
  • Có bản tóm tắt và audio sẵn.
  • Dùng tốt cho demo và onboarding.

Sách người dùng upload

User upload PDF, EPUB, TXT hoặc nhập nội dung. Backend tạo job xử lý bằng AI rồi đưa sách vào thư viện cá nhân của user.

  • Mặc định là private.
  • Có trạng thái xử lý rõ ràng.
  • AI tạo tóm tắt và audio riêng.
  • Chat AI chỉ dựa trên nội dung sách đó.
Quyết định sản phẩm Một bảng Books có thể chứa cả sách curated và sách user upload. Phân biệt bằng SourceType, Visibility và OwnerUserId. Cách này giúp frontend thư viện vẫn đồng bộ, không cần tách thành hai hệ thống khác nhau.

3. Kiến trúc backend đề xuất

Backend nên tiếp tục dùng ASP.NET Core MVC hiện tại. Không cần tách frontend thành SPA riêng ở giai đoạn đầu. Cách tốt nhất là thêm service layer, EF Core, Identity và các background job xử lý AI. 

Dia Browser
  -> ASP.NET Core MVC/Razor
    -> Controllers
      -> Services
        -> Entity Framework Core
          -> PostgreSQL

AI Processing
  -> Upload file
  -> Create ProcessingJob
  -> Extract text
  -> Chunk content
  -> Summarize with AI
  -> Generate audio with TTS
  -> Mark book as Ready
  -> User reads/listens/chats

Stack chính

  • Backend: ASP.NET Core MVC.
  • Database: PostgreSQL.
  • ORM: Entity Framework Core.
  • Authentication: ASP.NET Core Identity.
  • Background jobs giai đoạn đầu: IHostedService hoặc queue đơn giản trong database.
  • Background jobs nâng cấp: Hangfire hoặc Quartz.NET.
  • File storage giai đoạn đầu: wwwroot/uploads.
  • File storage production: S3, Cloudflare R2 hoặc Supabase Storage.
  • AI summarization: tích hợp LLM qua service riêng.
  • Text to speech: TTS provider qua service riêng.

5. Pipeline AI xử lý sách

Đây là phần cốt lõi của LemonInk. Khi user upload sách, backend không chỉ lưu file mà phải tạo một quy trình xử lý nhiều bước. Mỗi bước cần có trạng thái để frontend hiển thị loading, pending, error hoặc ready.

01 Upload User upload PDF/EPUB/TXT.
02 Extract Backend trích xuất text.
03 Chunk Chia nội dung thành đoạn.
04 Summarize AI tạo tóm tắt.
05 Audio TTS tạo audio.
06 Ready User đọc, nghe, chat.

Trạng thái cần hiển thị trên frontend

UploadedĐã nhận file
ExtractingTextĐang đọc file
SummarizingAI đang tóm tắt
GeneratingAudioĐang tạo audio
ReadyCó thể đọc/nghe/chat
FailedCần retry hoặc báo lỗi

Luồng upload sách của user

  1. User chọn file và bấm upload.
  2. Backend validate file: dung lượng, định dạng, quyền user.
  3. Tạo record Book với SourceType = UserUpload, Visibility = Private, ProcessingStatus = Uploaded.
  4. Lưu file vào storage và tạo BookFile.
  5. Tạo ProcessingJob loại ExtractText.
  6. Background worker lấy job và xử lý.
  7. Trích xuất text xong thì tạo chunk.
  8. Tạo job SummarizeBook.
  9. AI tạo summary, key takeaways, sections.
  10. Tạo job GenerateAudio.
  11. TTS tạo audio từ bản tóm tắt.
  12. Cập nhật book thành Ready.
  13. User nhận thông báo sách đã sẵn sàng.
Điểm cần cẩn thận Upload và AI processing có thể mất thời gian. Không nên xử lý toàn bộ trong request upload. Request upload chỉ nên lưu file và tạo job. Việc extract, summarize, audio phải chạy background.

6. Service layer

Service Nhiệm vụ Giai đoạn
IBookService Lấy sách public, sách private của user, chi tiết sách, search/filter/sort. Nền tảng

IAudioGenerationService Gọi TTS để tạo audio từ summary. Audio
IProcessingJobService Quản lý queue, retry, trạng thái job. Infrastructure
IChatService Trả lời câu hỏi theo nội dung sách, lưu lịch sử chat. LemonAI
IBookmarkService Lưu/xóa bookmark, lấy danh sách đã lưu. User library
IReadingProgressService Lưu tiến độ đọc/nghe của user. User library

8. Quy trình triển khai backend

Giai đoạn 1: Nền tảng database và auth

Ưu tiên cao nhất
  1. Cài PostgreSQL local.
  2. Cài EF Core và Npgsql.
  3. Tạo ApplicationDbContext.
  4. Thêm ASP.NET Core Identity.
  5. Tạo user thật, login/register thật.
  6. Seed role Admin và Reader.
  7. Thay cookie mock bằng Identity.

Giai đoạn 2: Kho sách có sẵn

Curated library
  1. Tạo entity sách, category, summary, chapter, takeaway, audio.
  2. Seed 12 sách hiện có.
  3. Thay dữ liệu mock ở trang chủ, thư viện, trang đọc.
  4. Admin có thể tạo/sửa/publish sách curated.
  5. Thư viện hiển thị sách public.

Giai đoạn 3: Upload sách của user

User uploads
  1. Thêm trang upload sách.
  2. Validate file PDF/EPUB/TXT.
  3. Lưu file và tạo Book private.
  4. Tạo ProcessingJob.
  5. Hiển thị sách đang xử lý trong thư viện cá nhân.
  6. Cho user retry nếu xử lý lỗi.

Giai đoạn 4: AI tóm tắt

Summarization
  1. Trích xuất text từ file.
  2. Chia text thành chunk.
  3. Tạo prompt tóm tắt tổng quan.
  4. Tạo prompt tóm tắt theo chương hoặc section.
  5. Tạo key takeaways.
  6. Lưu summary vào database.
  7. Cập nhật trạng thái SummaryReady.

Giai đoạn 5: Tạo audio

Text to speech
  1. Lấy bản tóm tắt đã tạo.
  2. Chuẩn hóa text cho TTS.
  3. Gọi provider TTS.
  4. Lưu file audio.
  5. Lưu metadata audio vào DB.
  6. Cập nhật book thành Ready.

Giai đoạn 6: LemonAI chat theo sách

Contextual chat
  1. Chat request gửi kèm bookId.
  2. Backend kiểm tra quyền xem sách.
  3. Tìm chunk liên quan.
  4. Gửi context cho AI.
  5. Trả lời và lưu lịch sử.
  6. Hiển thị citation nếu có.

9. Roadmap theo giai đoạn

Giai đoạn Mục tiêu Kết quả cần đạt
Tuần 1 PostgreSQL, EF Core, Identity. User đăng ký/đăng nhập thật, database chạy ổn.
Tuần 2 Curated library. Trang chủ, thư viện, trang đọc lấy sách public từ DB.
Tuần 3 Bookmark, progress, thư viện cá nhân. User lưu sách, tiếp tục đọc, xem sách riêng.
Tuần 4 Upload sách. User upload file, thấy trạng thái xử lý.
Tuần 5 AI tóm tắt. File upload được extract và tạo summary tự động.
Tuần 6 Tạo audio. Summary được chuyển thành audio để nghe.
Tuần 7 LemonAI chat. Chat trả lời theo nội dung sách public và private.
Tuần 8 Admin và hoàn thiện. Admin quản lý sách, job, lỗi xử lý, thống kê.

11. Kết luận

Kế hoạch backend đúng cho LemonInk phải xoay quanh mô hình hybrid library và AI processing pipeline. Kho sách có sẵn giúp người dùng khám phá ngay, còn upload cá nhân mới là điểm làm sản phẩm khác biệt: AI đọc sách, tóm tắt, tạo audio và cho phép chat theo nội dung.

Thứ tự làm tốt nhất là: database + auth, curated library, user library, upload, AI summarization, audio generation, LemonAI chat, admin. Đi theo thứ tự này sẽ giúp dự án chuyển từ frontend demo sang fullstack thật mà không bị vỡ kiến trúc.
